Hero: preserve the stage, use it as the visual grammar.

This is now the reference quality: large product scale, warm white stage, clear CTA hierarchy, and a character that feels charming without making the app look childish.

Why it works

  • One huge product scene instead of a small framed illustration.
  • The phone, Pip, and CTA all reinforce one action: check before spending.
  • Warm light and open space make the brand feel calm and premium.

Carry forward

  • Use stage lighting and scale in the remaining major sections.
  • Keep text blocks compact; let visuals do more explanatory work.
  • Prefer one dominant object over several small images.

Implementation

  • Freeze hero layout except small polish and asset optimization.
  • Create shared section-stage CSS utilities from the hero patterns.
  • Use hero mobile spacing as the acceptance bar for all later sections.
Acceptance: hero remains at 90+ while the rest of the page adopts its scale, calm, and clarity.
Build packet

Lock the current hero as the regression baseline. Any later work must preserve desktop composition, two-line headline, mobile CTA clearance, and the transparent product asset.

Implementation order

1. Systemize the hero craft

Create shared stage, glow, product-art, and mobile spacing primitives before touching individual sections.

2. Upgrade the three biggest lifts

Start with Ask Pip, Anti-budget/Meet Pip, and Balance because they have the largest score upside.

3. Convert the proof sections

Refine How it works, Pricing/Trust, and Blog so each section proves one thing visually.

4. Finish with conversion

Bring the Final CTA up last after the new visual system is available to reuse cleanly.

5. Mobile acceptance pass

Check every section at 320, 360, 390, 430, and 768 widths for overlap, dead space, and hierarchy.

6. Score again

Re-score each section against the 90/100 rubric and do one polish pass before shipping.

Risk controls and ship gates

These are the non-negotiable checks before this redesign ships. They exist because the hero took several iterations to get right, and the rest of the page should not repeat the same overlap, spacing, and image-quality mistakes.

Risk Trigger Mitigation Gate
Hero regression Shared CSS primitives accidentally change the current hero. Snapshot hero before each major section pass and keep hero-specific overrides isolated. Desktop and 390px mobile hero checks pass after every phase.
Generated-art drift New images introduce wrong palette, blurry UI text, or off-brand Pip shapes. Use generated images as direction unless individually approved as production assets. Every production image has a visual QA pass and file-size check.
Mobile overlap Large staged images collide with CTAs, headings, or adjacent sections. Top-anchor mobile images when they follow interactive controls; reserve canvas height explicitly. Check 320, 360, 390, 430, 477, and 768 widths.
Performance drag Nine richer sections add too many heavy PNGs. Convert production assets to optimized formats and use responsive sizes. Build passes and key marketing assets stay lazy-loaded outside the hero.
Trust-copy inaccuracy Pricing or security language claims more than the product supports. Reuse existing pricing constants and keep security proof tied to read-only data. Marketing tests assert pricing copy and public-language boundaries.
