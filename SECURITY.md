# Security Policy

Pip handles sensitive financial context. Please report suspected vulnerabilities privately.

## Reporting

Use GitHub's private vulnerability reporting when available, or contact the repository maintainers through a private channel.

Do not open public issues for vulnerabilities, credentials, personal financial data, authentication bypasses, payment bugs, or provider-token handling.

## Scope

In scope:

- Authentication/session handling
- Supabase RLS or service-role misuse
- Plaid token handling
- Stripe billing or webhook entitlement bugs
- OpenAI/agent data leakage
- Account deletion or data export/deletion flaws

Out of scope:

- Marketing copy issues
- Social engineering
- Denial-of-service without data access impact
